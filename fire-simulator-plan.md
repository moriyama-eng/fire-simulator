# 実装計画: シード値固定と対数t分布モデルの追加

## 1. 目的 (Objective)
FIRE モンテカルロ・シミュレータに対し、以下の機能を追加し、ユーザーがより精緻かつ再現性のある検証を行えるようにする。
- 乱数生成アルゴリズムを `Xoshiro128**` に変更し、シード値を固定・再現可能にする機能。
- 幾何ブラウン運動（対数正規分布）に加え、現実の金融市場で発生するファットテールリスク（暴落）を考慮できる「対数t分布モデル」を追加する機能。

## 2. 対象ファイル (Key Files)
- `index.html`: UIコンポーネント（シード値入力、変動モデル選択、自由度設定、サマリカード、画像出力用レイアウト）の追加。
- `js/app.js`: `Xoshiro128**`の実装、乱数シード管理、対数t分布モデルの計算ロジック、UIイベントの制御。

## 3. UI/UXの仕様と提案 (Proposed Solution)

### A. 乱数シード値の固定
- **配置場所**: 左カラムの一番下、「シミュレーション実行」ボタンの直上。
- **仕様**:
  - シード値のテキスト入力欄と「固定する(ON/OFF)」のトグルボタンを配置。
  - トグルON時: 入力欄の値（数値）をシードとして使用。空欄や無効な値の場合は新規生成。
  - トグルOFF時: 常に現在時刻等から新規シードを生成。
  - 実行後: 実際に使用されたシード値を入力欄に反映し、次回以降も同じシードで再現できるようにする。
- **サマリカードへの反映**: 既存のパラメータグリッドの最後に「新しい行」を追加し、「乱数シード値」として表示。

### B. 対数t分布モデルの追加
- **配置場所**: 左カラム「マーケット設定」カードの最上部. 
- **仕様**:
  - 項目名「リスク資産の変動モデル」。
  - セレクトボックスにて以下の2つの選択肢を用意する。
    1. **対数正規分布 (伝統的な金融理論)**
    2. **対数t分布 (現実的なリスク管理)**
    ※初心者にも直感的でモデルの意図が伝わりやすい素晴らしいネーミング提案のため、そのまま採用します。
- **自由度（df）の扱い**:
  - トグルボタンで「自動」と「固定」を切り替え可能とする。
  - 「自動」選択時は、入力欄をグレーアウト（無効化）し、内部でボラティリティに応じた値を計算する。
  - 「固定」選択時は、入力欄を有効化し、ユーザーが自由度を直接指定可能。
- **「自動」時の自由度の計算・外挿仕様（承認済）**:
  - ボラティリティ10%でdf=5、30%でdf=3となる線形補間式: `df = 5 - 0.1 * (Vol - 10)`
  - **10%未満の場合**: `df = 5` に固定（クランプ）。
  - **30%超の場合**: `df = 3` に固定（クランプ）。
- **サマリカードへの反映**: パラメータグリッド内の「現金バッファ設定」の直上に、1行全体を使って以下のように表示。
  - 項目名: `変動モデル`
  - 表示内容: `対数正規分布` または `対数t分布（自由度: 4.0）`
- **画像保存 (Capture) への反映**: 出力画像の「月間取崩し」の右隣（空欄部分）に以下のように配置。シード値は画像に出力しない。
  - 表示内容: `変動モデル: 対数正規分布` または `変動モデル: 対数t分布（自由度: 4.0）`
  - **絵文字の提案**: `🌊`（波：市場の「変動」やうねりを視覚的に表すため）を採用します。

## 4. 実装ステップ (Implementation Steps)

1. **数学関数の追加・修正**
   - `app.js` の `mulberry32` を `xoshiro128ss` (Xoshiro128**) に置き換え。
   - 初期シードから4つの32bit状態変数を生成する `splitmix32` を追加。
   - `randomNormal` に加え、t分布の乱数を生成する `randomT(rngFunc, df)` を追加（標準正規分布とカイ二乗分布から生成）。

2. **UIコンポーネントの追加 (`index.html`)**
   - 「マーケット設定」の最上部に変動モデル選択（セレクトボックス）と自由度（自動/固定トグル＋数値入力）UIを追加。
   - 実行ボタンの直上にシード値（入力欄＋固定トグル）UIを追加。
   - 画像保存用コンテナ（`captureContainer`）の該当空欄に変動モデル表示用の要素（`capModel`、絵文字 `🌊`）を追加。

3. **パラメータ取得とイベント連動 (`app.js`)**
   - `getParams()` に新しい設定値（モデル種類、自由度トグル、自由度手動値、シード固定トグル、シード値）の取得処理を追加。
   - 変動モデル切り替え時や自由度トグル切り替え時に、入力欄の有効/無効化やボラティリティ変更に伴う「自動計算値」のリアルタイムプレビューを実装。

4. **シミュレーションエンジンの改修**
   - トグルOFFまたは空欄なら `Date.now()` 等でシード生成し、UI入力欄へセット。
   - パス生成ループ内で、変動モデルが「対数t分布」の場合、月次リターンの生成において `randomT` で生成した乱数をボラティリティスケールに補正して使用（t分布の分散 `df/(df-2)` に基づくスケーリング）。

5. **サマリカードと画像出力の改修**
   - `updateSummaryCard` および `renderEmptySummaryCard` に、合意したレイアウト（現金バッファ設定の上に変動モデル、最後にシード値）の通りUI出力を追加。
   - `saveImage()` 処理内で `capModel` に変動モデルの設定文字列を転写。

## 5. GitHub リリース・運用ワークフロー (GitHub Workflow)

本機能追加はシステムの根幹に関わる重要なアップデートであるため、以下のプロセスでバージョン管理およびリリースを実施する。

1. **作業ブランチ**:
   - ブランチ名: `feature/advanced-sim-models` （高度なシミュレーションモデルとシード固定機能の追加）
2. **コミット・Squash戦略**:
   - 作業中の試行錯誤コミットはすべて最後に `git reset --soft` で取り消し、1つの綺麗なコミットに集約（Squash）する。
   - コミットメッセージ案:
     ```text
     feat: 対数t分布モデルおよび乱数シード値固定機能の追加

     - Xoshiro128**アルゴリズムによる乱数生成とシード値の保存・固定機能を実装
     - 幾何ブラウン運動に加え、ファットテールリスクを再現する対数t分布モデルを追加
     - UIに変動モデル選択（対数正規分布/対数t分布）と自由度設定（自動/固定）を追加
     - サマリカードおよび画像保存機能へ変動モデル・シード値を反映
     ```
3. **Pull Request (PR) 計画**:
   - **タイトル**: `feat: 対数t分布モデルと乱数シード固定機能の追加`
   - **本文 (Description)**:
     - **概要**: 乱数シード値によるシミュレーションの再現性確保と、より現実的な暴落リスクを評価できる対数t分布モデルを導入。
     - **主な変更点**: (上記コミットメッセージと同等の詳細を記載)
     - **UI/UX変更**: マーケット設定の最上部に変動モデル、実行ボタン上にシード値を配置。画像保存時のレイアウト更新。

   - マージ後、`gh release create --draft` を用いてドラフトを作成。
   - **リリース Title**: `v1.5.0: ファットテール対応（対数t分布）とシード値固定機能`

     * **対数t分布モデルの追加**: 従来の「対数正規分布（伝統的な金融理論）」に加え、「対数t分布（現実的なリスク管理）」を選択できるようになりました。これにより、リーマンショック級の暴落（ファットテールリスク）を考慮した、より厳しめのFIRE達成確率をシミュレーション可能です。
     * **乱数シード値の固定機能**: 実行ボタンの直上にシード値設定を追加しました。シミュレーション結果を完全に再現・保存することが可能になりました。
     * **自由度の自動調整**: 対数t分布選択時の「自由度」を、設定されたボラティリティに応じて自動で最適化する機能（手動固定も可能）を搭載しました。
     * **サマリ・画像出力の拡張**: シミュレーション結果のサマリカードと保存用画像（Xシェア用）に、選択した変動モデルとシード値が表示されるようになりました。

## 6. テストと検証 (Verification)
- シード固定ONで同じシードを入力した場合、何度実行してもサマリ数値（成功率・最終総資産等）およびグラフ形状が完全に一致することを確認。
- シード固定OFFの場合、実行ごとにシード入力欄の値が更新され、結果がばらつくことを確認。
- 変動モデルで「対数t分布」を選んだ際、同一ボラティリティ設定の「対数正規分布」と比べ、最大ドローダウンの発生確率（ファットテール）が深く（悪く）なる傾向があるか確認。
- 自由度の自動補間が正しく機能しているか（Vol=10%で5、Vol=30%で3、それ以外でクランプ）をブラウザ上で確認。
- サマリカードの表示レイアウト崩れがないか確認。
- 「画像を保存」機能が正常に動作し、指定位置に `🌊 変動モデル: 対数t分布（自由度: 4.0）` のように印字され、シード値が含まれないことを確認。
